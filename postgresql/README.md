# PostgreSQL Stack

The `/postgresql` stack installs PostgreSQL inside the Kubernetes cluster using
the Bitnami PostgreSQL Helm chart. It creates a dedicated namespace, generates
the PostgreSQL admin password, stores it in a Kubernetes Secret, and publishes
remote-state outputs for downstream stacks.

This stack is designed for the learning and test environment created by the
root Terraform stack. It is not a production database architecture.

## 🧭 What it creates

- Kubernetes namespace `postgresql` by default
- Generated PostgreSQL admin password
- Kubernetes Secret `postgresql-auth` by default
- Bitnami PostgreSQL Helm release named `postgresql`
- Remote-state outputs consumed by `/postgresql/n8n_db`

## ✅ Prerequisites

Before applying this stack:

1. Apply the root stack and wait for the cluster to finish bootstrapping.
2. Confirm `kubeconfig` exists in the repository root.
3. Confirm `backend.hcl` exists in the repository root and points to the OCI
   Object Storage backend used by the root stack.
4. Authenticate with the OCI CLI using the `DEFAULT` profile:

   ```bash
   oci session authenticate
   ```

The apply scripts read the root backend settings, export the required
remote-state variables, initialize this stack, and run `terraform apply`.

## 🚀 Create

Run from the repository root.

Linux/macOS:

```bash
./apply-postgresql.sh
```

Windows PowerShell:

```powershell
.\ApplyPostgresql.ps1
```

After apply, PostgreSQL is available inside the cluster through the Kubernetes
service created by the Helm chart in the `postgresql` namespace.

## 🛑 Destroy

Destroy dependent stacks first, especially `/postgresql/n8n_db` and `/n8n`.
Then run:

Linux/macOS:

```bash
./destroy-postgresql.sh
```

Windows PowerShell:

```powershell
.\DestroyPostgresql.ps1
```

## ⚙️ Configuration

Common variables are defined in `variables.tf`:

- `kubeconfig_path`: path to the cluster kubeconfig, defaulting to `../kubeconfig`
- `postgresql_namespace`: namespace where PostgreSQL is installed
- `postgresql_secret_name`: Secret that stores the generated admin password

The scripts provide the remote-state variables automatically from the root
`backend.hcl` and `state.tf`.

## ⚠️ Notes

- The admin password is generated by Terraform and stored in Kubernetes; do not
  hardcode database passwords in documentation or values files.
- Storage behavior is controlled by the Helm chart values and the cluster's
  available StorageClass.
- This stack should be applied before `/postgresql/n8n_db`.
